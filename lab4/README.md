Лабораторна робота №4

1. Мета роботи

- **Порахувати кількість повних тетрад** у двійковому поданні десяткового числа.  
- **Побудувати ієрархію класів із використанням шаблону Factory Method (Virtual Constructor)**.  
- **Розширити відображення результатів текстовою таблицею**, параметри якої задає користувач.  
- **Продемонструвати принципи ООП**: перевизначення (overriding), перевантаження (overloading), поліморфізм (dynamic method dispatch).  
- **Забезпечити діалоговий інтерфейс із користувачем**.  
- **Підготувати javadoc-документацію** до основних класів.

2. Постановка задачі

1. Взяти за основу вихідний текст проекту лабораторної роботи, що обчислює кількість повних тетрад у двійковому представленні десяткового числа.  
2. Використовуючи шаблон проектування **Factory Method**, розширити ієрархію класів похідними, які відображають результат:
   - у вигляді звичайного тексту;
   - у вигляді **текстової таблиці**, параметри якої (ширина стовпців, символи рамки тощо) задає користувач.
3. Продемонструвати:
   - **заміщення (overriding)** методів у похідних класах від загального інтерфейсу відображення;  
   - **перевантаження (overloading)** методів (наприклад, різні варіанти обчислень/відображення з різними параметрами);  
   - **динамічне призначення методів (поліморфізм)** через роботу з посиланнями базового типу (`ResultDisplay`, `DisplayFactory`).  
4. **Організувати діалоговий інтерфейс**: користувач задає число, обирає спосіб відображення (звичайний текст / таблиця, параметри таблиці).  
5. **Розробити клас для тестування** основної функціональності (перевірка правильності обчислень, різних варіантів відображення).  
6. Забезпечити **коментарі у форматі javadoc** для автоматичної генерації документації.

3. Структура проєкту (основні класи)

- `CalculationData` – зберігає:
  - вихідне десяткове число;
  - його двійкове подання;
  - кількість повних тетрад.  
- `TetradCalculator` – виконує обчислення:
  - перетворення десяткового числа в двійкове;
  - підрахунок кількості повних тетрад.  
- `ResultDisplay` – інтерфейс відображення результатів (поліморфна точка розширення).  
- `TextResultDisplay` – реалізація `ResultDisplay` для звичайного текстового виведення.  
- `TableResultDisplay` (розширення) – реалізація `ResultDisplay`, яка виводить дані у вигляді **текстової таблиці** з параметрами, що задає користувач.  
- `DisplayFactory` – інтерфейс фабрики для створення об’єктів `ResultDisplay`.  
- `TextDisplayFactory` – фабрика для створення `TextResultDisplay`.  
- `TableDisplayFactory` (розширення) – фабрика для створення `TableResultDisplay` з урахуванням параметрів таблиці.  
- `CalculationManager` – керування колекцією результатів:
  - збереження результатів в `ArrayList`;  
  - серіалізація/десеріалізація у файл (`data.dat`).  
- `Main` – діалоговий інтерфейс із користувачем:
  - введення числа;
  - вибір типу відображення (текст / таблиця);
  - налаштування параметрів таблиці;
  - запуск обчислень і відображення результатів.  
- `Test` – тестування основної функціональності:
  - перевірка обчислення кількості тетрад для різних вхідних чисел;
  - перевірка різних реалізацій `ResultDisplay` та фабрик.

4. Використані технології та прийоми

- **ООП**: інтерфейси, наслідування, поліморфізм, перевизначення та перевантаження методів.  
- **Патерн проєктування Factory Method**:
  - дозволяє відокремити логіку створення об’єктів від їх використання;
  - спрощує додавання нових способів відображення результатів без зміни існуючого коду.  
- **Колекції Java (`ArrayList`)** – зберігання множини результатів.  
- **Серіалізація** – збереження/відновлення колекції результатів у файл (`data.dat`).  
- **Javadoc** – коментарі для автоматичної генерації документації.

5. Приклад роботи програми (текстовий режим)

Для вхідного числа `255` можливий результат:

- Десяткове число: `255`  
- Двійкове число: `11111111`  
- Кількість повних тетрад: `2`  
- Стан колекції результатів збережено у файл `data.dat`.

У випадку таблиці результат може виводитись у вигляді текстової таблиці, де користувач задає, наприклад, ширину стовпців та тип рамки.

6. Запуск проєкту

1. Перейти в директорію проєкту:
   - `cd lab3`  
2. Скомпілювати всі `.java` файли:
   - `javac *.java`  
3. Запустити головний клас:
   - `java Main`  
4. Дотримуватись підказок у консолі:
   - ввести число;  
   - обрати тип відображення;  
   - (за потреби) ввести параметри таблиці.

7. Генерація документації javadoc

Для генерації документації (у каталозі `doc`) можна використати команду:

```bash
javadoc -d doc *.java
```

Після цього документацію можна переглядати через файл `index.html` у каталозі `doc`.
